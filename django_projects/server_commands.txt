Если вы нажали Ctrl + Z в терминале Ubuntu, то процесс сервера был остановлен и перемещен в фоновый режим (остановлен, но не завершен). В результате, когда вы пытаетесь запустить сервер повторно, может возникнуть конфликт с уже запущенным процессом.

Для восстановления сервера и его правильного запуска, вы можете выполнить следующие шаги:

Введите команду jobs для просмотра списка заданий в фоновом режиме. Вы увидите номер задания, связанный с остановленным сервером.

jobs

Восстановите задание в активное состояние с помощью команды fg (от англ. foreground) и номера задания. Например, если номер задания равен 1, введите fg %1. Это вернет процесс сервера в передний план и возобновит его работу.

fg %<номер процесса>

Если восстановление задания не сработало или вы все еще не можете запустить сервер, можно попробовать принудительно завершить процесс сервера. Для этого выполните команду ps -ef | grep "имя_процесса" для поиска процесса сервера. Затем найдите ID процесса сервера и выполните команду kill -9 <ID_процесса> для его принудительного завершения. После этого вы сможете запустить сервер заново.

ps -ef | grep "имя_процесса"  # номер процесса стоит вторым по порядку
kill -9 <ID_процесса>

Обратите внимание, что Ctrl + Z используется для приостановки процесса в терминале. Если вам нужно остановить сервер, лучше использовать комбинацию клавиш Ctrl + C, которая отправляет сигнал завершения (SIGINT) процессу и корректно останавливает сервер.
